# 系统设置移植 - 群聊视觉上下文与多模态转述 (astrbot_plugin_sys_setting_port)

这是一个为 AstrBot 提供群聊上下文感知、图片转述、自定义艾特等待和私聊主动回复的插件。

## 🌟 核心功能

AstrBot 的图片处理和群聊上下文感知分别由不同链路负责。本插件提供按模型、按群聊白名单控制的图片处理，并把群友未直接唤醒 Bot 时的聊天内容临时注入当前请求。

本插件提供以下高级特性：

1. **完整群聊视觉上下文**：白名单群里的普通文字、At、引用和图片都会按原始消息顺序记录；图片在原位置替换为描述，例如 `[小月/20:30]: 看看这个 [图片：一只橘猫]`。
2. **正常时序注入**：最近的群聊感知条目会在每次 LLM 请求时作为一次性的 User 上下文，插入正式对话历史之后、当前唤醒消息之前；不会写入正式对话数据库，也不会累计到下一轮。当前触发消息会从群聊感知列表中排除，避免和当前 prompt 重复。
3. **严格消息淘汰**：按群聊消息条数保留最近记录，超过上限从最旧消息删除；一条多图消息仍只计一条。
4. **图片描述复用**：图片描述缓存只用于后续引用时避免重复识别，支持容量上限和 TTL，到期自动失效；缓存本身不会单独注入上下文。
5. **多模态直通与插件兼容**：普通群聊图片只以描述进入群聊时间线。引用图片默认只发送已有描述；当前消息命中“仔细看”“看原图”“看细节”“重新看”“重看”“再看”等可配置关键词时，多模态模型才会额外收到原图。图生图等插件仍可获取引用原图；纯文本模型始终只收到描述。
6. **引用来源保留**：引用图片会记录被引用消息的原发送者，避免 A 引用 B 的图片后被误判为 A 发图。
7. **智能重试与兜底**：主转述模型失败时自动重试，并可切换到备用模型。
8. **精准触发**：群聊视觉上下文只处理配置白名单中的群，不会读取其他群的闲聊图片。
9. **自定义等待机制 (增强版)**：接管系统原有的「只 @ 机器人」等待逻辑，支持自定义等待时间、自定义等待回复词。
10. **严格用户隔离**：在群聊中，只有真正 @ 机器人的那个用户发送的下一条消息才会被捕获，完美避免其他群友的闲聊乱入！
11. **完美超时唤醒**：超时后若用户未发消息，强制唤醒 AI 主动找话题，并且若有其余插件引用功能， AI 回复会**正确引用**用户之前发送的那条 @ 消息，体验丝滑原生！
12. **私聊主动回复 (AGI 进化版)**：当私聊闲置时，机器人会主动找用户聊天。**核心优势**：
   - **纯净上下文架构**：直接调用 LLM 并手动触发钩子，完美触发所有下游插件（如 `nova_omni` 的思维链拦截与分段、表情包插件等），不会将冗长的系统提示词污染到数据库的历史记录中！
   - **记忆无缝衔接**：主动发出的消息会正确写入 AstrBot 的历史记录数据库，AI 绝不“精神分裂”，完美记住自己刚刚找过你。
   - **双模式驱动**：支持“纯随机模式”和“智能判定模式”。智能模式下，Bot 会调用轻量级模型“察言观色”，根据聊天氛围决定是否主动开口。
   - **极致人文关怀**：支持免打扰时段（半停机不耗资源）、白名单机制、未回复次数上限，以及动态渲染的高级提示词！
   - **硬盘级持久化**：主动回复的状态（闲置时间、未回复次数等）会安全地保存在 `data/plugin_data` 目录下，即使重启 AstrBot 或重载插件，时间线也绝对不会丢失！

## ⚙️ 配置说明

请在 AstrBot 的 WebUI 后台 -> 插件管理 -> `astrbot_plugin_sys_setting_port` 中进行配置：

### ⏳ 等待机制配置
- **启用自定义等待机制**：开启后接管系统的等待逻辑。
- **自定义等待时间 (秒)**：只 @ 机器人后，等待用户发送下一条消息的超时时间。
- **等待期间是否回复** / **自定义等待回复词**：等待期间机器人发送的提示文本。
- **超时后是否强制唤醒**：超时后若用户未发消息，直接唤醒 AI 主动回复。

### 💬 主动回复配置
- **允许主动回复的用户列表**：白名单机制，只有填入此列表的 QQ 号/用户 ID 才会获得主动消息服务。
- **主动回复触发模式**：选择“纯随机模式”或“智能判定模式”。
- **智能判定模型 / 提示词**：仅智能模式有效，用于判定当前氛围是否适合主动聊天。
- **最小 / 最大触发间隔 (分钟)**：控制主动消息的频率和保底触发时间。
- **免打扰时段 (24小时制)**：例如 `23-7`，在此期间插件进入半停机状态，不发送消息也不调用判定模型。
- **最大未回复次数上限**：防止无限骚扰，达到上限后暂停主动回复。
- **主动回复触发提示词**：支持 `{{current_time}}` 和 `{{unanswered_count}}` 占位符，指导大模型根据情境动态调整语气。

### 👥 群聊视觉上下文配置
- **启用插件群聊视觉上下文**：启用后由本插件完整替代 AstrBot 原生群聊上下文感知，记录普通文字、At、引用及图片描述。
- **群聊视觉上下文白名单**：仅处理列表中的群号/群 ID，留空则不处理群聊。
- **群聊图片理解模型**：群友发图时使用的视觉模型。
- **群聊图片理解提示词**：生成群聊历史图片描述的提示词。
- **群聊感知最大消息数**：每群保留的最近消息条数，超出后删除最旧消息；该历史包含普通文字，不只是图片描述。
- **图片描述复用时长 / 缓存上限**：仅用于引用旧图时复用已有描述；过期或超出容量后重新识别，不影响群聊历史条数。
- **引用图片重新查看关键词**：仅多模态模型生效。当前消息命中关键词时，引用图片会在已有描述之外再次作为原图发送；纯文本模型始终不会收到原图。

启用本插件群聊视觉上下文的群后，请关闭 AstrBot 原生的「启用群聊上下文感知」和「自动理解图片」，否则会重复记录普通群聊和重复调用图片模型。本插件会完整记录普通文本与图片描述，不是只保留图片缓存。

### 🖼️ 多模态转述配置
- **多模态转述模型**：选择用于看图的主力模型（如 `openai`, `gemini` 等）。
- **兜底次选转述模型**：主模型失败时的备用模型。
- **需要转述的模型列表**：填入需要被转述的纯文本模型名称（如 `gpt-3.5-turbo`, `deepseek-v3`）。**支持模糊匹配**，只要当前模型名称包含列表中的关键词（不区分大小写）即可触发转述。
- **图片转述提示词**：自定义发送给多模态模型的看图指令。
- **最大重试次数** / **触发重试的关键词**：控制重试逻辑。

## ⚠️ 注意事项

1. **关闭系统转述**：使用本插件前，请务必将 AstrBot 系统设置（LLM 配置）中的「默认图片转述模型」留空！
2. **关闭系统等待**：如果您启用了本插件的「自定义等待机制」，请务必在系统设置（平台设置）中关闭原有的「只 @ 机器人是否触发等待」！
